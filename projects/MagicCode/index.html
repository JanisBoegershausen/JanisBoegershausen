Magic Code What is this project?
This project currently goes under the development name "Magic Code" and so far, it's just a prototype of a node-based spell making system.
How it works
The system is based on two essential parts. Spells and the spell-nodes they contain.
Each spell contains a list of spell-nodes, of which one is saved as the starting node.

Every type of spell-node (AddForce, Multiply, etc.) is its own class, that derives from one SpellNode class. That way every type of node has its own code and data, but they can all be handled the same by the spell script. Nodes store a list of sockets, which store their type, current data and the node they are connected to.

An important distinction is the one between active and passive nodes.
Active nodes are the ones with green sockets (order-of-execution). Each of them run their code once called and then run the next node they are connected to.
On the other hand, we have passive nodes. These are nodes that just handle data in some way (like all the math functions for example). They don't have this order-of-execution socket, because they are run every time another node tries to read data from one of its data-sockets. This also propagates backward, so if an Add-Force-Node tries to read data from a multiply-node, the multiply node is run, but if there are other passives plugged into the multiply, they are run first and so on.

So when a spell is cast, the start node calls the node that is connected to its order-of-execution-socket, which then does its thing, reads data to the input sockets, writes data to its output sockets and then calls the next node, connected to its order-of-execution-socket.
